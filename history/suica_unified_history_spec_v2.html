Transaction row — core fields (always visible in list)

SF: 種別 column
 Icon + color per transaction type. Replaces the current text-only 種別 column (入/出, 物販, カード, バス等). Lets users scan without reading each row. Must Icon + label, not icon alone. Color must not be the only differentiator (accessibility).

SF: 月日 column
 Date and time of transaction. Current app shows date only (月日). Adding time allows users to reconstruct their day accurately. Must Group rows by date with a sticky date header. Show relative time ("2時間前") for same-day items.

 For transit: "新宿 → 渋谷". For e-money: real merchant name (e.g. "NewDays 新宿東口") — not 物販. For charge: "クレジットカード チャージ". For points: reason (e.g. "乗車ポイント" or "ポイント交換"). Must Merchant name improvement is already live in the current app. Points label is new.

 Yen amount (−¥405) for SF transactions. Point amount (＋12pt) for JRE POINT events. Use red for deductions, green for additions. Show unit clearly (¥ vs pt). Must Mixed units (¥ and pt) in one list — always show the unit label, never a bare number.

 Suica balance remaining after this transaction. For JRE POINT rows, show point balance after. Two balance types — display the one relevant to the row type. Must Current app shows this as 残高差額 (balance + diff in one cell). Split into two distinct fields for clarity.
Transaction types — all types from SF screen + new unified types
JR rail transit
鉄道乗車
SF: 入 / 出
 IC gate tap-in (入) and tap-out (出). Show as a single merged row: "新宿 → 渋谷" with fare. Current SF screen splits this into two rows which is hard to read. Must Merge 入/出 into one row in the redesign. Show off-peak discount (オフピーク割引) if applied.
e-money purchase
電子マネー利用
SF: 物販
 Suica payment at stores, vending machines, kiosks. Show real merchant name. If merchant name is unavailable, fall back to 物販 — do not show blank. Must Merchant name already live in current app per function table rank 4.
Credit card charge
クレジットチャージ
SF: カード / モバイル
 Manual top-up via linked credit card. Amount added shown as positive (e.g. ＋¥5,000). Source label: "カードチャージ" or "オートチャージ" to distinguish manual vs automatic. Must Distinguish manual charge vs auto-charge — different triggers, user needs to know which fired.
Bus & non-JR transit
バス・他社交通
SF: バス等
 Suica tap on buses and non-JR operators (e.g. Toei, private railways, community buses). Current SF shows operator code (e.g. "フジエク", "神奈中", "西東京B"). Redesign should show operator name in full where possible. Must New Was missing from previous spec. Seen in every screenshot — common transaction type.
Special fare / Shinkansen
新幹線・特殊入場
SF: ＊入
 The ＊ prefix in current SF history flags special entry: Shinkansen IC boarding, inter-area transfers, or surcharge routes. Redesign should surface the reason clearly rather than a cryptic asterisk. Must New Was missing from previous spec. ＊ is unclear to users — label it explicitly (e.g. "新幹線 自由席").
JRE POINT earned
ポイント獲得
SF: —（not shown）
 Points earned are shown as a sub-row attached directly below their triggering transaction — not as a separate floating entry. This makes the cause-and-effect visible: "I rode → I earned."

Trigger rules:
・JR rail ride → 乗車ポイント (ride points, typically 6pt per ¥200 fare)
・e-money purchase at partner store → 購買ポイント (purchase points)
・View Card payment (credit card charge event) → カード利用ポイント
・Off-peak ride → オフピークポイント (bonus on top of base ride points)
・Commuter pass purchase → 定期券購入ポイント

If points arrive with a processing delay (batch), the sub-row appears as "pending" (獲得予定) until confirmed. 		Must New Sub-row pattern keeps the list clean while showing points in context. Pending state handles JRE batch delay gracefully.
JRE POINT redeemed
ポイント交換・利用
SF: —（not shown）
 Points redeemed appear as a standalone row (no parent transaction) since redemption is a deliberate user action, not a side-effect of another event.

Trigger rules:
・ポイントでSuicaチャージ → shows −Xpt and ＋¥X charge received
・グリーン券をポイントで購入 → shows −Xpt and ticket detail
・JRE MALL / partner reward → shows −Xpt and reward name 		Must New Standalone because redemption is user-initiated, not triggered by a Suica transaction.
Commuter pass purchase
定期券購入・更新
SF: 定期（rare）
 Purchase or renewal of a commuter pass. Infrequent but high value when it occurs. Show route, validity period, and amount paid. Should Distinct icon (pass card). Renewal reminder could link from here.

SF: —（app purchase）
 In-app purchase of a green car ticket. Show route, travel date, amount (¥ or pt). Links to function table rank 5. Should Show whether paid in yen or JRE POINT.
Coupon / ticket use
クーポン・チケット利用
SF: —（not shown）
 When a limited-time coupon or ticket was redeemed. Show coupon name, merchant, discount amount saved. Should Reinforces value — user can see cumulative savings over time.
Filtering controls
Type filter chips
種別フィルター
 Horizontal scrollable chips: All / 鉄道 / バス / お買い物 / チャージ / ポイント / 定期・チケット. Defaults to All. Filter Chip row is standard on iOS. No equivalent in current app.
Date range filter
期間フィルター
 Presets: 今週 / 今月 / 3ヶ月. Custom date picker for expense reporting use cases. Show monthly spend total when a monthly filter is active. Filter Monthly total at top of filtered view is useful for household budgeting.
Card selector

 Toggle between Suica cards if user has multiple (e.g. phone + Apple Watch). Defaults to primary card. Filter Ties to multi-card carousel (function table rank 19).
Keyword search

 Search by station name or merchant name. Find a specific transaction quickly. Could Post-v1. Not in current app.
Detail view — shown when user taps a row
Full route & time
乗車詳細
 Full station names, tap-in/tap-out time, line name, fare breakdown. Useful for expense claims. Detail Current SF shows truncated station names (e.g. "地麻布十" for 麻布十番). Full name in detail view.
Points earned on this trip
この取引で獲得したポイント
 JRE POINT earned from this specific transaction. Only shown if points were earned. Reinforces the loyalty loop. Detail Links to JRE POINT balance screen.
Discount applied

 Off-peak discount, coupon applied, commuter pass coverage. Show original fare vs discounted fare. Detail Off-peak discount is increasingly common — makes the saving visible.
Export / receipt
明細出力
 Share or export individual transaction as PDF or image for expense submission. Could Post-v1. High value for business users.
✓ Resolved: JRE POINT earned rows attach to their triggering transaction as a sub-row. Pending state (獲得予定) handles batch processing delay. Redemption rows are standalone since they are user-initiated, not triggered by a Suica event.

⚑ Remaining open question: Should the points sub-row be visible by default on every row, or collapsed and only shown when the user taps the parent? Always-visible is better for discoverability but makes the list denser for commuters who don't care about points.
